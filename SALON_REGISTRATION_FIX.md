# Fix for Salon Registration RLS Error

## Problem
You're getting this error when trying to register a salon:
```
Error inserting salon: {
  code: '42501',
  details: null,
  hint: null,
  message: 'new row violates row-level security policy for table "salons"'
}
```

## Root Cause
The Row-Level Security (RLS) policies in your Supabase database are blocking anonymous salon registration. The current policy requires that `auth.uid()` matches the salon's `id`, but during registration:

1. There's no authenticated user session
2. The salon ID is generated by the database
3. There's no way for `auth.uid()` to match a new salon's ID

## Solutions

### Option 1: Update Database RLS Policies (Recommended)

Run the SQL migration file I created in your Supabase SQL Editor:

1. Go to your Supabase project dashboard
2. Navigate to SQL Editor
3. Copy and paste the contents of `database/fix-salon-registration-rls.sql`
4. Click "Run"

This will add policies that allow:
- Anonymous users to register new salons with 'pending' status
- Service creation during salon registration
- Existing salon owners to manage their salons

### Option 2: Add Service Role Key (Alternative)

Add the Supabase service role key to your environment variables:

1. In your Supabase project dashboard, go to Settings → API
2. Copy the "service_role" key (not the anon key)
3. Add to your `.env.local` file:
   ```
   SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here
   ```
4. Restart your development server

The updated code will automatically use the service role for salon registration, which bypasses RLS entirely.

### Option 3: Disable RLS Temporarily (Not Recommended for Production)

If you need a quick fix for development:

```sql
-- Temporarily disable RLS on salons table
ALTER TABLE salons DISABLE ROW LEVEL SECURITY;
ALTER TABLE services DISABLE ROW LEVEL SECURITY;
```

**Warning**: Only use this for development. Re-enable RLS before going to production.

## Verification

After applying the fix, test the salon registration:

1. Go to `/salon` on your frontend
2. Click "Fillo Regjistrimin" 
3. Complete the registration form
4. Submit the form

You should see a success message instead of the RLS error.

## Security Notes

- The recommended approach (Option 1) maintains security while allowing registration
- New salons are created with 'pending' status and require admin approval
- Only pending salons can be created anonymously
- Active salons still require proper authentication
